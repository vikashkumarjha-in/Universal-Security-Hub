# Contributing to Universal-Security-Hub

Thank you for your interest in contributing to the Universal-Security-Hub! We welcome all contributions that help improve functionality, security, and usability.

## Contribution Workflow

1. **Fork the Repository:** Create your own copy of `https://github.com/vikashkumarjha-in/Universal-Security-Hub`.
2. **Create a Feature Branch:** Always work on a separate branch (e.g., `git checkout -b feature/your-feature-name`).
3. **Commit Changes:** Keep your commits concise and descriptive.
4. **Push and Pull Request:** Push your changes to your fork and submit a Pull Request (PR) to the main repository.

## Standards
* **Code Style:** Ensure Python code in `src/` follows standard PEP 8 formatting.
* **Documentation:** If you add a new module, you must update `docs/usage.md` accordingly.
* **Security:** Do not include sensitive information, API keys, or academic-specific private details in your commits.

## Reporting Issues
If you encounter a bug, please open an issue in the repository with a detailed description, reproduction steps, and screenshots if applicable.
